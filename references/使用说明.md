# 智改数转报告生成 Skill 使用指南

## 唯一生成方式

本 skill 只允许使用官方模板原位填充：

```text
企业素材
  ↓
AI 提取并生成 JSON 数据
  ↓
scripts/run_report.sh
  ↓
scripts/remote_generate.py 上传 JSON 与图片附件
  ↓
远程服务校验授权并调用服务端生成器
  ↓
服务端生成器打开官方模板、原位填充并返回 .docx
  ↓
客户端保存完整报告
```

禁止输出 Markdown，禁止让用户复制粘贴，禁止新建空白 Word。

## 官方模板

官方模板由远程服务持有并作为唯一版式基准：

```text
zhigai-report-server/assets/智改数转调研表-20260507v1.0.docx
```

公开客户端不读取、不修改模板；每次生成都由服务端在原模板上原位填充。

## 运行入口

统一使用：

```bash
bash ~/.codex/skills/zhigai-report/scripts/run_report.sh \
  --data 生成的数据.json \
  --output 企业名称_智改数转报告_日期.docx
```

`run_report.sh` 的职责只有两项：

1. 选择可用的 Python 解释器并调用 `scripts/remote_generate.py`。
2. 将授权码、JSON、图片附件和输出路径传给远程客户端。

授权失败、附件缺失或服务端生成失败时立即停止；不得切换到本地生成或 Markdown 转 Word。

## 企业总体情况与建设方案图片

如用户提供图片，AI 需要按用户说明转成四个图片槽位和一个成熟度图片字段：

```text
第一部分：插入在（二）企业总体情况文字后、主要产品之前
第二部分：插入在主要产品文字之后、产品工艺之前
第三部分：插入在产品工艺之后、（三）企业智改数转情况之前
第四部分：替换三、企业智改数转建设方案中的“总体技术架构图”占位图
成熟度图片：插入在“智能制造成熟度现状评测具体情况”下方、“智能制造成熟度现状分析”上方
```

例如用户说明“第1-3张图片为第一部分插入，第4张为第二部分插入，第5-7张为第三部分插入，第8张为第四部分架构图，第9张为成熟度图片”，则 JSON 中应保持上传顺序写入：

```json
{
  "企业总体图片": {
    "第一部分": ["第1张图片路径", "第2张图片路径", "第3张图片路径"],
    "第二部分": ["第4张图片路径"],
    "第三部分": ["第5张图片路径", "第6张图片路径", "第7张图片路径"],
    "第四部分": ["第8张图片路径"]
  },
  "智能工厂对标分析": {
    "成熟度评分表已上传": true,
    "成熟度评分图片": "第9张图片路径"
  }
}
```

企业总体情况中的“主要产品”和“产品工艺”应优先使用列表结构：

```json
{
  "企业总体情况分段": {
    "总体文字": "企业概况文字...",
    "主要产品列表": [
      "三类医疗器械：一次性使用输液器、一次性注射器。",
      "二类医疗器械：配药器、医用外科口罩。"
    ],
    "产品工艺列表": [
      {"名称": "一次性输液器", "流程": "原料入厂检验→PP 颗粒干燥→注塑→...→成品仓储出库"},
      {"名称": "一次性注射器 / 配药器", "流程": "医用 PP 原料检验→注塑成型→...→外包装赋 UDI 入库。"}
    ]
  }
}
```

脚本会自动把 `主要产品列表` 渲染成“（2）主要产品”标题和 `(1)(2)(3)(4)` 分项段落；把 `产品工艺列表` 渲染成“（3）产品工艺”标题和“编号产品名称 + 下一段流程”。不要把产品和工艺压缩成一整段，也不要省略这两部分的语义标题。

图片路径必须是本地可读取的绝对路径；缺失图片不得静默跳过。

## 智能工厂对标分析

“企业智改数转痛点需求分析 / （一）智能工厂对标分析”必须优先使用结构化字段 `智能工厂对标分析`。

用户可能提供三类图片：

```text
图1：政府系统导出的智能制造成熟度评分图
图2：能力要素维度分析表
图3：核心维度对标分析表
```

处理规则：

1. 图1用于提取成熟度总分、能力要素权重/得分、能力子域得分，并插入到“智能制造成熟度现状评测具体情况”下方。
2. 图2用于生成并填充模板中的“能力要素 / 权重 / 得分 / 现状分析”表。
3. 图3用于生成并填充模板中的“核心维度 / 2025版基础级标准要求 / 企业现状匹配情况”表。
4. 智能工厂等级判定必须结合成熟度得分、现场现状和《智能工厂梯度培育要素条件（2025年版）》；达不到基础级时写“低于基础级”或“数字化萌芽阶段”。
5. 缺少官方得分时写“得分待补充”或提示用户补充，不得编造评分。
