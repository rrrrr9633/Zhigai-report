# 智改数转报告生成 Skill 安放与使用教程

本文面向最终使用者，从压缩包解压开始，说明如何安放 skill、准备 Python 环境、安装依赖、处理依赖不足导致的失败，以及如何使用专用提示词生成报告。

## 1. 解压压缩包

假设你拿到的压缩包名称类似：

```text
zhigai-report.zip
```

解压后应看到一个目录：

```text
zhigai-report/
├── SKILL.md
├── requirements.txt
├── agents/
├── assets/
├── examples/
├── references/
└── scripts/
```

必须确认以下文件存在：

```text
zhigai-report/SKILL.md
zhigai-report/assets/智改数转调研表-20260507v1.0.docx
zhigai-report/scripts/run_report.sh
zhigai-report/scripts/setup_env.sh
zhigai-report/scripts/generate_report.py
zhigai-report/requirements.txt
```

如果缺少 `assets/智改数转调研表-20260507v1.0.docx`，报告无法按基准模板生成。

## 2. 安放 skill 目录

### macOS / Linux

推荐放到：

```bash
mkdir -p ~/.codex/skills
mv /解压后的路径/zhigai-report ~/.codex/skills/zhigai-report
```

安放完成后的目录应为：

```text
~/.codex/skills/zhigai-report/SKILL.md
~/.codex/skills/zhigai-report/assets/智改数转调研表-20260507v1.0.docx
~/.codex/skills/zhigai-report/scripts/run_report.sh
```

如果你使用的是 Cursor 自定义 skills 目录，也可以按你的环境放入对应 skills 目录；核心要求是整个 `zhigai-report` 文件夹必须完整保留，不能只复制 `SKILL.md`。

### Windows

推荐放到用户目录下的 skills 文件夹，例如：

```text
C:\Users\你的用户名\.codex\skills\zhigai-report
```

目录结构同样必须完整保留：

```text
C:\Users\你的用户名\.codex\skills\zhigai-report\SKILL.md
C:\Users\你的用户名\.codex\skills\zhigai-report\assets\智改数转调研表-20260507v1.0.docx
C:\Users\你的用户名\.codex\skills\zhigai-report\scripts\generate_report.py
```

## 3. 安装 Python

本 skill 使用 Python 脚本直接填充 Word 模板，必须安装 Python 3。

### 3.1 检查是否已有 Python

macOS / Linux：

```bash
python3 --version
```

Windows：

```powershell
python --version
```

看到类似下面的输出即可：

```text
Python 3.10.13
```

推荐 Python 版本：Python 3.9 或更高。

### 3.2 macOS 安装 Python

如果没有 Python，推荐使用 Homebrew：

```bash
brew install python
python3 --version
```

如果没有 Homebrew，可先安装 Homebrew，或直接从 Python 官网下载安装包：

```text
https://www.python.org/downloads/
```

### 3.3 Ubuntu / Debian 安装 Python

```bash
sudo apt-get update
sudo apt-get install -y python3 python3-venv python3-pip
python3 --version
```

### 3.4 CentOS / Fedora / RHEL 安装 Python

Fedora / 新版 RHEL：

```bash
sudo dnf install -y python3 python3-pip
python3 --version
```

CentOS / 旧版 RHEL：

```bash
sudo yum install -y python3 python3-pip
python3 --version
```

### 3.5 Windows 安装 Python

1. 打开 Python 官网：

```text
https://www.python.org/downloads/windows/
```

2. 下载 Python 3.x 安装包。
3. 安装时必须勾选：

```text
Add python.exe to PATH
```

4. 安装完成后打开 PowerShell：

```powershell
python --version
pip --version
```

如果提示 `python 不是内部或外部命令`，通常是安装时没有加入 PATH，需要重新安装并勾选 `Add python.exe to PATH`。

## 4. 安装 Python 依赖

本 skill 目前依赖：

```text
python-docx
```

### 4.1 推荐方式：让脚本自动安装

通常不需要手工安装依赖。首次生成报告时，统一运行：

```bash
bash ~/.codex/skills/zhigai-report/scripts/run_report.sh \
  --data /你的路径/data.json \
  --output /你的路径/输出报告.docx
```

如果首次失败，`run_report.sh` 会自动执行：

```bash
~/.codex/skills/zhigai-report/scripts/setup_env.sh
```

自动完成：

```text
检查 Python
创建 .venv 虚拟环境
升级 pip
安装 requirements.txt 中的依赖
重新生成报告一次
```

### 4.2 手动安装依赖

如果你想提前安装依赖，可执行：

```bash
cd ~/.codex/skills/zhigai-report
python3 -m venv .venv
.venv/bin/python -m pip install --upgrade pip -i https://pypi.tuna.tsinghua.edu.cn/simple --trusted-host pypi.tuna.tsinghua.edu.cn
.venv/bin/python -m pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple --trusted-host pypi.tuna.tsinghua.edu.cn
```

Windows PowerShell：

```powershell
cd C:\Users\你的用户名\.codex\skills\zhigai-report
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install --upgrade pip -i https://pypi.tuna.tsinghua.edu.cn/simple --trusted-host pypi.tuna.tsinghua.edu.cn
.\.venv\Scripts\python.exe -m pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple --trusted-host pypi.tuna.tsinghua.edu.cn
```

## 5. Python 或依赖不足导致失败怎么办

### 5.1 报错：未检测到 Python

典型提示：

```text
未检测到 Python。
```

处理方式：

1. 按第 3 节安装 Python 3。
2. 安装后重新打开终端。
3. 运行：

```bash
python3 --version
```

4. 再重新生成报告。

### 5.2 报错：No module named docx

典型提示：

```text
ModuleNotFoundError: No module named 'docx'
```

说明 `python-docx` 没装好。处理方式：

```bash
cd ~/.codex/skills/zhigai-report
bash scripts/setup_env.sh
```

然后重新运行生成命令。

### 5.3 报错：pip 下载失败或超时

可能是网络访问 PyPI 不稳定。脚本默认使用清华镜像：

```text
https://pypi.tuna.tsinghua.edu.cn/simple
```

如果仍失败，可换镜像：

```bash
cd ~/.codex/skills/zhigai-report
PIP_INDEX_URL=https://mirrors.aliyun.com/pypi/simple PIP_TRUSTED_HOST=mirrors.aliyun.com bash scripts/setup_env.sh
```

### 5.4 报错：python3-venv 不存在

Ubuntu / Debian 常见。处理方式：

```bash
sudo apt-get update
sudo apt-get install -y python3-venv python3-pip
cd ~/.codex/skills/zhigai-report
bash scripts/setup_env.sh
```

### 5.5 报错：Permission denied

如果脚本没有执行权限，不要直接运行 `./scripts/run_report.sh`，改用：

```bash
bash ~/.codex/skills/zhigai-report/scripts/run_report.sh \
  --data /你的路径/data.json \
  --output /你的路径/输出报告.docx
```

如需修复权限：

```bash
chmod +x ~/.codex/skills/zhigai-report/scripts/*.sh
```

### 5.6 自动修复后仍失败

如果看到：

```text
自动修复后仍然生成失败。请把上方完整错误信息发给我，我会继续定位。
```

请把以下内容发给技术支持或 AI：

```text
1. 运行的完整命令
2. 终端完整报错
3. data.json 路径
4. 输出路径
5. Python 版本：python3 --version
6. 当前系统：macOS / Windows / Linux
```

不要改用 Markdown 转 Word，不要新建空白 Word。这个 skill 的正确路线始终是“官方模板原位填充”。

## 6. 图片放在哪里、怎么描述

报告中图片分为五类输入：企业总体情况三处图片、第三部分架构图、成熟度评分图片。

### 6.1 第一部分图片

插入位置：

```text
一、（二）企业总体情况
总体文字之后、主要产品列表之前
```

用途：企业外观、厂区、车间概况、营业/资质/总体环境等能支撑企业总体情况的图片。

JSON 字段：

```json
"企业总体图片": {
  "第一部分": ["/absolute/path/to/image-1.png"]
}
```

### 6.2 第二部分图片

插入位置：

```text
一、（二）企业总体情况
主要产品列表之后、产品工艺列表之前
```

用途：主要产品、产品包装、产品线、核心产品展示图。

JSON 字段：

```json
"企业总体图片": {
  "第二部分": ["/absolute/path/to/image-2.png"]
}
```

### 6.3 第三部分图片

插入位置：

```text
一、（二）企业总体情况
产品工艺列表之后、（三）企业智改数转情况之前
```

用途：生产工艺、设备现场、流程图、车间关键工序图片。

JSON 字段：

```json
"企业总体图片": {
  "第三部分": ["/absolute/path/to/image-3.png"]
}
```

### 6.4 第四部分图片

插入位置：

```text
三、企业智改数转建设方案
（一）xx公司智改数转规划方案
3．三年智改数转规划方案
(1)总体方案架构
总体技术架构图
```

用途：总体技术架构图、系统架构图、平台架构图、数字化建设蓝图。

推荐 JSON 字段：

```json
"智改数转建设方案": {
  "总体技术架构图": ["/absolute/path/to/architecture.png"]
}
```

也兼容写在：

```json
"企业总体图片": {
  "第四部分": ["/absolute/path/to/architecture.png"]
}
```

如果两个位置都写了，以 `智改数转建设方案.总体技术架构图` 为准。

### 6.5 成熟度图片

插入位置：

```text
二、企业智改数转痛点需求分析
（一）智能工厂对标分析
(2)智能制造成熟度现状评测具体情况 下方
(3)智能制造成熟度现状分析 上方
```

用途：政府系统导出的成熟度评分截图、成熟度雷达图、评分总览图。

JSON 字段：

```json
"智能工厂对标分析": {
  "成熟度评分表已上传": true,
  "成熟度评分图片": "/absolute/path/to/maturity-score.png"
}
```

注意：只有用户明确提供成熟度评分表/评分截图时，才允许填写成熟度分数、阶段判断、能力要素得分等内容。

## 7. 发布给使用者的专用提示词

把下面这段提示词复制给 AI 使用。方括号里的内容按实际情况补充。

```text
使用 智改数转报告生成 写一份丹东制药企业的智改数转诊断报告。

基准模板：智改数转调研表-20260507v1.0.docx。

本次目标企业的调研素材如下：
【在这里粘贴工商信息、企业基本情况、主营业务、主要产品、生产工艺、设备清单、软件系统、网络建设、平台建设、安全措施、AI 应用、成熟度得分、痛点、改造需求、预算约束等素材】

图片插入说明如下：
第一部分图片：第【填写图片序号，如 1-3】张，插入到“企业总体情况”总体文字之后、主要产品之前。
第二部分图片：第【填写图片序号，如 4】张，插入到主要产品之后、产品工艺之前。
第三部分图片：第【填写图片序号，如 5-7】张，插入到产品工艺之后、“企业智改数转情况”之前。
第四部分图片：第【填写图片序号，如 8】张，用于替换“三、企业智改数转建设方案”中的“总体技术架构图”。
成熟度图片：第【填写图片序号，如 9】张，插入到“智能制造成熟度现状评测具体情况”下方、“智能制造成熟度现状分析”上方。

生成要求：
1. 必须基于内置官方模板原位填充并输出 .docx 文件，不要输出 Markdown 报告。
2. 企业基本信息表保留给用户手工填写。
3. 成熟度评分、能力要素得分、能力子域得分必须逐项依据我提供的成熟度图片或评分素材，不得编造。
4. 设备价格、软件价格、供应商、厂商所在地没有明确素材时，分别写“待补充”“待调研”。
5. 第三部分“企业智改数转建设方案”按企业实际需求生成。模板中括号里的“例”只是参考，不要照抄；PLM、MES、WMS、ERP、供应商协同、EMS、BI、AI 等系统按企业需求增删。
6. “三年智改数转规划方案”必须填入模板预留表格。
7. “具体改造项目”数量和类别按企业需求动态生成，不强制套用 8 个示例项目；每个项目必须包含项目名称、项目内容、改造环节、建设目标、预计投入、投资回报周期、实施步骤。
8. 图片路径必须使用本地可读取的绝对路径；如果图片缺失，先停止并提示，不要静默跳过。
```

## 8. 最小运行检查

安装和安放完成后，可用示例数据测试：

```bash
bash ~/.codex/skills/zhigai-report/scripts/run_report.sh \
  --data ~/.codex/skills/zhigai-report/examples/test_data.json \
  --output /tmp/智改数转测试报告.docx
```

成功后应看到：

```text
✅ 报告生成完成！
```

并得到：

```text
/tmp/智改数转测试报告.docx
```

如果测试通过，说明 skill、模板、Python、依赖都已经可用。
