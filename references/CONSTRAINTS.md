# 完整约束规则详解

## 一、AI职责边界（明确不越界）

### AI负责填充的部分

1. **一、（二）企业总体情况**
   - 格式：分段正文；包含总体文字、主要产品、产品工艺
   - 字数：300字以内
   - 内容：主营业务、竞争优势、发展现状与规划、主导产品、主要生产工艺
   - 主要产品必须作为第二部分呈现；产品工艺必须作为第三部分呈现，不得只写列表而缺少语义标题

2. **一、（三）企业智改数转情况**
   - 1. 总体情况：使用“公司总体概述”五项结构
     - 主营业务
     - 主导产品
     - 主要工艺
     - 竞争优势
     - 数字化智能化现状与规划
   - 2. 数字化智能化设备应用情况：整体情况描述 + 四项能力描述 + 关键装备统计表
     - 设备数控化、自动化水平概述
     - 设备自动化控制方面
     - 设备互联方面
     - 设备监控与决策方面
   - 3. 网络化建设情况（文字段落）
   - 4. 平台建设情况（文字段落）
   - 5. 工业软件和信息化系统应用情况（文字描述 + 关键软件统计表）
   - 6. 网络和数据安全情况（文字段落）
   - 7. 人工智能应用情况（文字段落）
   - 8. 其他情况（文字段落）

3. **二、企业智改数转痛点需求分析**
   - （一）智能工厂对标分析：成熟度现状、梯度等级、核心维度与要素差距
   - （二）企业痛点分析：行业规范与企业特点总述 + 6 个企业实际痛点小节，其中至少 1 项为行业特有核心管控短板

4. **三、企业智改数转建设方案**
   - （一）企业智改数转规划方案
     - 1. 建设思路
     - 2. 建设目标：总体目标 + 短期/中期/长期具体目标
     - 3. 三年智改数转规划表
     - 4. 总体方案架构与建设内容
     - 5. 三年三阶段总体实施计划表
     - 6. 五类项目预期成效
   - （二）具体改造项目：数量与类别按企业真实需求动态生成，不套用固定示例项目

### AI不处理的部分（严禁越界）

1. **一、（一）企业基本信息表** → 用户手工填写
2. **三、（三）服务商推荐** → 已固定内容，AI不修改
3. **三、（四）企业诊断后服务计划及方案** → 已固定内容，AI不修改

## 二、绝对禁止编造的内容清单

### 数字类（无素材一律填"待补充"）

- 单套软件价格（万元）
- 具体投资金额（可用区间估算，如"50-100万元"）
- 企业年营收精确数字（可用区间，如"5000万~1亿元"）

### 信息类（无素材一律填"待调研"或"待补充"）

- 设备供应商名称
- 软件供应商名称
- 原厂商所在地（国外/国内，省外/省内）
- 统一社会信用代码
- 产品批准文号
- 资质证书编号

### 评分类（必须由用户提供官方数据）

- 智能制造成熟度总分及分项得分
- 智能工厂等级评定结果
- 任何政府官方系统导出的评分数据

**处理原则**：
- 如用户未提供成熟度评分，明确告知"需从政府官网系统获取官方评分数据"
- 如用户未提供设备价格，表格中填写"待补充"
- 如用户未提供供应商信息，表格中填写"待调研"

## 三、表格填充规范

### 表1：关键装备统计表

**列名**（不可修改）：
- 关键装备种类
- 名称
- 规格/型号
- 系统状态
- 供应商
- 原厂商所在地（国外/国内，省外/省内）

**填充规则**：

1. **关键装备种类**（从以下选择）：
   - 高档数控机床与工业机器人
   - 增材制造装备
   - 智能传感与控制装备
   - 智能检测与装配装备
   - 智能物流与仓储装备
   - 行业成套装备

2. **系统状态**：
   - 在用
   - 停用（必须写明原因，如"功能不符"、"故障待修"等）

3. **供应商**：
   - 有素材：填写实际供应商名称
   - 无素材：填写"待调研"

5. **原厂商所在地**：
   - 格式："国外/德国" 或 "国内/省外/江苏" 或 "国内/省内"

   - 新型软件（工业APP、低代码平台等）

2. **部署方式**（从以下选择）：
   - 本地部署
   - 私有云部署
   - 公有云部署
   - 混合云部署

3. **系统状态**：
   - 在用
   - 停用（必须写明原因）

4. **单套软件价格**：
   - 有素材：填写实际价格
   - 无素材：填写"待补充"

5. **供应商**：
   - 有素材：填写实际供应商名称
   - 无素材：填写"待调研"

## 四、文字内容字数控制

| 章节 | 字数要求 | 超出处理 |
|------|---------|---------|
| （二）企业总体情况 | 建议约500字，可适度浮动 | 优先保持“总体概述、主营业务、主导产品、主要工艺与设备、竞争优势”完整，不因超过建议字数阻断生成或强制压缩 |
| 设备应用情况 | 整体描述及四项能力描述应结合已提供设备事实 | 覆盖数控/自动化、控制、互联、监控与决策 |
| 网络化建设情况 | 80-120字 | 控制在此范围 |
| 平台建设情况 | 60-100字 | 控制在此范围 |
| 软件应用情况描述 | 100-150字 | 控制在此范围 |
| 网络和数据安全情况 | 80-120字 | 控制在此范围 |
| 人工智能应用情况 | 60-100字 | 控制在此范围 |
| 每个改造项目描述 | 50-100字/项 | 言简意赅 |

**压缩技巧**：
- 删除形容词和修饰语
- 合并同类表述
- 使用专业术语缩写（如"工业互联网平台"→"工业互联网平台"）
- 避免重复描述

## 五、改造项目必备要素检查清单

每个按企业需求动态生成的改造项目必须包含以下要素：

| 序号 | 要素 | 说明 | 示例 |
|------|------|------|------|
| 1 | 项目名称 | 简明扼要 | "生产线自动化升级改造" |
| 2 | 项目内容 | 具体做什么 | "对关键数控车床、磨床增加数据采集网关并完善工控网络" |
| 3 | 改造环节 | 影响哪个环节 | "车削、磨削与质量检测工序" |
| 4 | 建设目标 | 可验收的功能结果 | "形成关键设备状态、工艺参数和质量数据的受控采集能力" |
| 5 | 预计投入金额和投资回报周期 | 可估算 | "预计投入80-100万元，回报周期18个月" |
| 6 | 实施步骤 | 分步骤描述 | "（1）设备选型与采购（2）现场安装调试（3）人员培训上线" |

**缺失要素判定为不合格**，必须补全。

## 六、行业话术适配规则

- 默认不预置任何行业专用术语，先从企业名称、主营业务、产品、工艺、客户要求和现场约束确认所属行业。
- 行业规范、监管要求、认证体系和专用系统必须有用户素材依据；无法确认时使用通用制造表述并标记待调研。
- 通用设备企业重点分析订单交付、设计变更、工艺标准、设备利用、计划排产、生产执行、质量检验、仓储物流和售后服务。
- 涉密或军工企业重点确认工控网与办公网边界、外网接入限制、跨网交换机制、本地化部署和数据分级授权；不得建议绕过隔离边界直接互联。
- 示例数据只用于展示字段结构，不得把示例中的行业、设备、系统、评分或建设项目迁移到其他企业报告。

## 七、预算约束自适应规则

根据企业预算情况，自动调整改造项目优先级：

### 预算 < 200万元（低预算）

**优先保留**：
- 工业软件应用（轻量化MES、数字化看板）
- 网络建设（工业以太网升级）
- 网络和数据安全（基础防护）

**推迟或删减**：
- 智能装备应用（重型设备改造）
- 平台建设（工业互联网平台接入）
- 人工智能应用（AI质检系统）

### 预算 200-500万元（中等预算）

**可包含**：
- 工业软件应用（完整MES+WMS）
- 网络建设（5G+工业以太网）
- 网络和数据安全（完整防护体系）
- 智能装备应用（局部产线自动化）

**选择性包含**：
- 平台建设（轻量化平台）
- 人工智能应用（单场景AI应用）

### 预算 > 500万元（高预算）

**全面建设**：
- 六大领域项目全部包含
- 每个项目可细化为多个子项目
- 可规划分阶段实施

## 八、成熟度评分解读规范

### 用户必须提供的数据

```
总体成熟度：X.X级
├── 人员与能力：X.X分
├── 技术与基础设施：X.X分
├── 资源与数据：X.X分
└── 制度与文化：X.X分
```

### AI解读方式

1. **识别短板维度**：找出得分最低的1-2个维度
2. **分析差距原因**：结合企业现状，说明为何该维度得分低
3. **提出改进方向**：给出针对性建议（50-80字）

**示例**：
```
企业成熟度总体为2.1级，处于规范级向集成级过渡阶段。其中"技术与基础设施"维度得分1.8分，为明显短板，主要原因是设备数字化率低、网络覆盖不足。建议优先推进设备联网改造和工业以太网升级，提升数据采集能力。
```

### 智能工厂等级判定

参考《智能工厂梯度培育要素条件（2025年版）》：

| 成熟度等级 | 智能工厂阶段 | 典型特征 |
|-----------|------------|---------|
| 1-2级 | 基础级 | 基础信息化，局部数字化 |
| 2-3级 | 先进级 | 数据互联互通，初步智能化 |
| 3-4级 | 卓越级 | 全面数字化，深度智能化 |
| 4-5级 | 领航级 | 数据驱动，自主决策 |

**输出格式**：
```
根据企业当前成熟度2.1级，判定为"基础级"智能工厂。建议在2026-2027年重点推进XXX、XXX改造，力争达到"先进级"标准。
```

## 九、最终输出规范

### 唯一交付物

最终交付物必须是基于官方模板原位填充生成的 `.docx` 文件。

生成链路固定为：

```
企业素材
  ↓
结构化 JSON 数据
  ↓
scripts/run_report.sh
  ↓
scripts/generate_report.py 打开 assets/智改数转调研表-20260507v1.0.docx
  ↓
在官方模板中填充正文和表格
  ↓
输出完整 .docx 文件
```

### 禁止输出方式

- 禁止把正文生成为 Markdown 后再转 Word。
- 禁止新建空白 Word 文档后写入正文。
- 禁止要求用户复制内容到官方模板。
- 禁止用 Markdown 表格替代模板内表格。

### 项目描述数据格式

JSON 中每个改造项目按以下字段组织，最终由脚本填入官方模板：

```
项目名称：关键加工设备数据采集改造
项目内容：对关键数控车床、磨床开展接口盘点，在受控工控内网部署数据采集网关
改造环节：车削、磨削与质量检测工序
建设目标：形成设备状态、停机原因、工艺参数和检测结果的受控采集能力
预计投入金额和投资回报周期：预计投入60-80万元（估算），投资回报周期18-24个月（估算）
实施步骤：（1）数据边界确认（2）设备接口盘点（3）试点接入（4）数据校验（5）验收推广
```

## 十、常见错误防范清单

| 错误类型 | 错误示例 | 正确做法 |
|---------|---------|---------|
| 编造价格 | "单台设备价格120万元"（无依据） | "单台设备价格：待补充" |
| 编造供应商 | "供应商：XX智能科技"（虚构） | "供应商：待调研" |
| 编造评分 | "计算得出成熟度2.3级" | "请从政府系统获取官方评分" |
| 超出字数 | 企业总体情况写了500字 | 压缩至300字以内 |
| 缺失要素 | 改造项目只写了项目名称和内容 | 必须包含6个要素 |
| 行业内容无依据 | 从历史样例复制行业监管或专用系统要求 | 仅使用企业素材可证明的行业要求，否则采用通用制造表述 |
| 越界填充 | 填写了企业基本信息表 | 该表由用户手填 |
| 修改固定内容 | 修改了服务商推荐表 | 该部分已固定，不可修改 |

## 十一、生成后自检流程

### 必检项（任意一项不通过，视为失败）

1. ✅ （二）企业总体情况是否≤300字
2. ✅ 两个表格（设备表、软件表）是否都已生成
3. ✅ 表格中是否存在编造的价格或供应商
4. ✅ 六个改造项目是否都包含6个必备要素
5. ✅ 是否编造了成熟度评分
6. ✅ 是否填充了（一）企业基本信息表（应由用户填）
7. ✅ 是否修改了（三）服务商推荐和（四）服务计划（应保持固定）

### 输出确认格式

```
✓ 自检通过：
  - 字数控制：企业总体情况 XXX字（≤300字）
  - 表格生成：设备表 X行，软件表 X行
  - 数据合规：无编造价格/供应商/评分
  - 要素完整：6个改造项目均包含必备要素
  - 边界清晰：未越界填充固定部分

输出文件：/path/to/企业名称_智改数转报告_日期.docx
```
